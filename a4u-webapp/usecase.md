# **📝 a4u (Apply4U) MVP 핵심 유즈케이스 시나리오**

**문서 버전:** v1.0

**작성 목적:** a4u MVP의 핵심 가치인 'AI 연동 기능'을 중심으로, 사용자와 시스템(AI), 관리자 간의 상호작용 절차를 정의합니다.

## **1\. 프론트엔드 (사용자 화면)**

### **UC-F-01: 기존 이력서 파싱 및 자동 맵핑 (AI)**

| 항목 | 내용 |
| :---- | :---- |
| **Use Case Name** | 기존 이력서 파싱 업로드 |
| **Actor** | 사용자(User), 시스템(AI) |
| **Description** | 사용자가 기존에 보유한 이력서 파일을 업로드하면, AI가 내용을 분석하여 빌더 폼에 맞게 자동 입력해 준다. |
| **\[접근 경로\]** | 대시보드 ➔ '새 이력서 작성' ➔ 제출 관리/시작 선택 (select.html) |
| **Step / Description** | 1\. 사용자가 '기존 문서 업로드' 영역에 PDF 또는 Word 파일을 드래그 앤 드롭하거나 탐색기로 선택하여 업로드한다. 2\. 화면에 Loading Spinner가 표시되며 "AI가 이력서를 분석 중입니다..." 문구가 노출된다. 3\. **\[AI 연동\]** 시스템은 LLM API를 호출하여 문서 내 텍스트를 인적사항, 경력, 스킬 항목으로 분류(JSON 파싱)한다. 4\. 시스템은 분류된 데이터를 이력서 빌더(builder.html)의 각 입력 폼에 맵핑하여 화면을 전환한다. 5\. 사용자는 자동으로 채워진 데이터를 확인하고 수정을 시작한다. |
| **예외 처리** | \- 파일 인식 실패 또는 AI API 지연(토큰 초과 등) 시, "텍스트 추출에 실패했습니다. 직접 양식을 채워주세요." 알림 팝업 후 빈 양식 빌더로 이동한다. |

### **UC-F-02: AI 코칭 어시스턴트 적용 (AI)**

| 항목 | 내용 |
| :---- | :---- |
| **Use Case Name** | 이력서 빌더 AI 문장 코칭 |
| **Actor** | 사용자(User), 시스템(AI) |
| **Description** | 사용자가 경력 사항을 짧은 단문으로 입력하면, AI가 STAR 기법에 맞춘 프로페셔널한 문장으로 다듬어 준다. |
| **\[접근 경로\]** | 이력서 빌더 화면 (builder.html) 내 경력 입력 폼 |
| **Step / Description** | 1\. 사용자가 경력 사항 폼에 "매출 20% 올림, 마케팅 캠페인 기획함"과 같이 짧은 텍스트를 입력한다. 2\. 사용자가 입력 필드 옆의 **\[✨ AI 코칭받기\]** 버튼을 클릭한다. 3\. 버튼이 Skeleton UI 또는 로딩 상태로 변경된다. 4\. **\[AI 연동\]** 시스템은 입력된 텍스트와 Few-shot 샘플 프롬프트를 조합하여 LLM API에 STAR 기법 문장 변환을 요청한다. 5\. 시스템은 AI가 생성한 1\~2개의 추천 문장을 툴팁 또는 하단 영역에 제안한다. 6\. 사용자가 제안된 문장 중 하나를 클릭(적용)하면, 텍스트 폼의 내용이 교체된다. 7\. 우측 '실시간 렌더링 미리보기' 패널에 변경된 내용이 즉시 반영된다. |

## **2\. 어드민 (관리자 화면)**

### **UC-A-01: AI 프롬프트 실시간 튜닝**

| 항목 | 내용 |
| :---- | :---- |
| **Use Case Name** | AI 프롬프트 실시간 튜닝 및 적용 |
| **Actor** | 관리자(Admin) |
| **Description** | 관리자가 코드 수정 및 배포 없이 프론트엔드에서 사용되는 AI 기능의 시스템 프롬프트를 수정하고 테스트한다. |
| **\[접근 경로\]** | 관리자 통합 대시보드 (admin.html) ➔ 좌측 LNB 'AI 프롬프트 관리 탭' |
| **Step / Description** | 1\. 관리자가 튜닝할 대상 프롬프트를 선택한다 (예: 이력서 파싱용, AI 코칭용, 스코어링용). 2\. 중앙 에디터 영역에 현재 적용 중인 Base System Prompt가 로드된다. 3\. 관리자가 에디터에서 프롬프트의 지시어, 제약사항, Few-shot 예시 등을 직접 수정한다. 4\. 우측 '테스트' 패널에 임의의 사용자 입력값(예: "개발자 3년차 경력")을 넣고 **\[테스트 실행\]** 버튼을 클릭한다. 5\. **\[AI 연동\]** 시스템은 수정된 프롬프트를 기반으로 LLM API를 호출하여 결과값을 화면에 출력한다. 6\. 관리자가 결과물을 확인하고 만족할 경우 **\[운영 환경 배포\]** 버튼을 클릭한다. 7\. 해당 시점부터 사용자의 프론트엔드 요청은 변경된 프롬프트를 기반으로 동작한다. |

### **UC-A-02: 자연어 기반 DB 관리 (Text-to-SQL)**

| 항목 | 내용 |
| :---- | :---- |
| **Use Case Name** | 자연어 to SQL 변환 및 실행 |
| **Actor** | 관리자(Admin), 시스템(AI) |
| **Description** | 관리자가 자연어로 DB 관련 명령을 내리면, AI가 이를 SQL문으로 변환하여 실시간으로 실행한다. |
| **\[접근 경로\]** | 관리자 통합 대시보드 (admin.html) ➔ 좌측 LNB 'AI 스키마 콘솔 탭' |
| **Step / Description** | 1\. 화면 상단에 현재 a4u.db (SQLite)의 테이블 및 컬럼 구조가 시각화되어 표시된다. 2\. 관리자가 하단 프롬프트 입력창에 "최근 3일간 가입한 유저의 이메일 목록을 뽑아줘"라고 자연어로 입력하고 전송한다. 3\. **\[AI 연동\]** 시스템은 자연어와 현재 DB 스키마 정보를 LLM에 전달하여 호환되는 SQL 쿼리를 생성한다. 4\. 변환된 SQL 코드(예: SELECT email FROM users WHERE created\_at \>= date('now', '-3 days');)가 화면에 노출된다. 5\. 관리자가 생성된 SQL을 검토한 후 **\[Execute(실행)\]** 버튼을 누른다. 6\. 시스템이 쿼리를 실행하고 하단 데이터 그리드 뷰에 결과값을 렌더링한다. |
