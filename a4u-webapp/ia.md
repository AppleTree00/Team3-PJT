# **🗺️ a4u (Apply4U) MVP Information Architecture (IA)**

**문서 버전:** v1.1 (AI 기능 명시 강화)

**작성 목적:** 3일 완성 MVP 시연을 위한 프론트엔드(사용자) 및 어드민(관리자) 환경의 페이지 구조와 실제 구현/미구현 기능, 그리고 AI 연동 포인트의 명확한 정의.

**범례:**

* 🟢 **\[구현\]**: 백엔드 API 연동 및 실제 작동하는 핵심 기능  
* 🟣 **\[AI 연동\]**: LLM(대형 언어 모델) API가 직접 호출되어 처리되는 핵심 지능형 기능  
* 🟡 **\[UI만\]**: 화면 이동 및 UI만 존재 (데이터 하드코딩)  
* 🔴 **\[미구현 팝업\]**: 클릭 시 handleUnavailableFeature() 호출 ("고도화 단계입니다" 안내)

## **1\. 프론트엔드 (사용자 화면 IA)**

*타겟 유저의 동선을 최소화하고 핵심 가치인 'AI 이력서 작성'으로 빠르게 유도합니다.*

### **1 Depth: 메인 & 인증**

* **메인 랜딩 (main.html)**  
  * 🟢 \[구현\] 메인 히어로 섹션 & 서비스 소개  
  * 🟢 \[구현\] '무료로 시작하기' 버튼 ➔ select.html 이동  
  * 🟢 \[구현\] '데모 보기' 버튼 ➔ dashboard.html 이동  
  * 🔴 \[미구현 팝업\] 풋터 링크 (이용약관, 개인정보처리방침 등)  
* **로그인 (login.html)**  
  * 🟢 \[구현\] 이메일/비밀번호 로그인 (세션 기반)  
  * 🔴 \[미구현 팝업\] 소셜 간편 로그인 (Google, 네이버, 카카오)

### **1 Depth: 메인 서비스 (GNB 네비게이션)**

* **대시보드 (dashboard.html)**  
  * 🟣 **\[AI 연동\] 이력서 스코어링:** 등록된 이력서의 키워드 및 완성도를 분석한 AI 종합 평가 점수 표시  
  * 🟢 \[구현\] 퀵 액션: '새 이력서 작성' / '이력서 보완하기' ➔ builder.html 이동  
  * 🟡 \[UI만\] 지원 현황 트래커 (Kanban 보드 \- 하드코딩 데이터)  
  * 🔴 \[미구현 팝업\] 지원 현황 '새로 추가', '필터' 버튼  
* **제출 관리 / 시작 선택 (select.html)**  
  * 🟢 \[구현\] 빈 양식으로 시작 ➔ builder.html 이동  
  * 🟣 **\[AI 연동\] 기존 이력서 파싱 업로드:** PDF/Word 업로드 시, AI가 텍스트를 분석하여 빌더의 인적사항/경력/스킬 폼(JSON 형태)으로 자동 맵핑 및 데이터 이관  
* **이력서 빌더 (builder.html) \- 🌟 MVP 핵심 화면**  
  * 🟢 \[구현\] 단일 대화/입력 폼 (인적사항, 경력, 스킬)  
  * 🟣 **\[AI 연동\] AI 코칭 어시스턴트:** 사용자가 입력한 짧은 경력 사항을 STAR 기법(상황-과제-행동-결과)에 맞춰 프로페셔널한 문장으로 다듬어주는 기능 (Few-shot 샘플 기반)  
  * 🟢 \[구현\] 실시간 이력서 렌더링 미리보기 (우측 패널)  
  * 🟢 \[구현\] 이력서 DB 저장 및 브라우저 내장 인쇄 (window.print())  
  * 🔴 \[미구현 팝업\] PDF 직접 다운로드, 외부 공유  
* **이력서 관리 (resume.html)**  
  * 🟢 \[구현\] 내 이력서 목록 조회 (샘플 3종 포함)  
  * 🟢 \[구현\] 목록에서 이력서 미리보기 및 인쇄 호출  
* **커리어 타임라인 (timeline.html)**  
  * 🟡 \[UI만\] 경력/학력 시각화 타임라인 렌더링

### **1 Depth: 유저 프로필 & 설정**

* **내 프로필 (profile-menu.html)**  
  * 🟢 \[구현\] 프로필 이미지 업로드 (실시간 Sync 및 브라우저 캐시 우회)  
  * 🟢 \[구현\] 기본 정보 수정 및 로그아웃  
  * 🔴 \[미구현 팝업\] 계정 삭제, 보안 설정 등

## **2\. 어드민 (관리자 화면 IA)**

*단일 SPA 구조 내에서 좌측 LNB를 통해 탭(Section)을 전환합니다.*

### **1 Depth: 관리자 통합 대시보드 (admin.html)**

* **개요 탭 / 회원 관리 탭 / 이력서 관리 탭 / 템플릿 관리 탭 / 파일 관리 탭**  
  * *(기존 명세와 동일하게 🟢 구현 상태 유지. 템플릿 관리는 물리 파일 업로드 및 브라우저 인쇄 호출 포함)*  
* **AI 프롬프트 관리 탭 (Prompts Section) \- 🌟 신규 추가**  
  * 🟣 **\[AI 연동\] 프롬프트 튜닝:** 프론트엔드에서 사용되는 '이력서 파싱', 'AI 코칭', '스코어링'의 Base System Prompt를 코드 배포 없이 실시간으로 수정하고 테스트하는 환경  
* **AI 스키마 콘솔 탭 (Schema Section)**  
  * 🟢 \[구현\] 현재 a4u.db (SQLite) 테이블/컬럼 구조 실시간 조회  
  * 🟣 **\[AI 연동\] 자연어 to SQL 변환:** "최근 3일간 가입한 유저 삭제해줘" 같은 자연어 입력 ➔ SQL DML/DDL 코드로 변환 및 즉시 적용(Execute) 기능

### **📝 팀원(에이전트) 작업 지침서**

1. **프론트엔드 (UI Agent) 주의사항:**  
   🔴\[미구현 팝업\] 처리 외에, 🟣\[AI 연동\] 파트 UI 작업 시 사용자 대기 시간(Loading Spinner, Skeleton UI)을 반드시 구현하여 AI 응답 지연 시 UX를 방어하십시오.  
2. **백엔드 (Dev Agent) 주의사항:**  
   AI 연동 파트(파싱, 코칭, SQL 변환)는 외부 LLM API(예: OpenAI)의 응답 지연 및 토큰 초과 오류(429, 500)에 대비한 예외 처리 로직(Fallback)을 반드시 포함해야 합니다.
