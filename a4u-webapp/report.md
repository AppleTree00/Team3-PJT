# [기술 보강] APPLY4U MVP 개발 보고서

## 1. 기술 스택 (Tech Stack)
* **Frontend**: HTML5, Tailwind CSS (반응형 UI), JavaScript (ES6+)
* **Backend**: Python / Flask (轻량 프레임워크 기반 API 서버)
* **Database**: SQLite (빠른 프로토타이핑 및 이식성)
* **Deployment/Tool**: VS Code, Git/GitHub, Tailwind CDN

## 2.1. PM 역할: 보고서 작성 및 문서 정합성 검증
* PM은 `pm.md`를 기준으로 `UI.md`, `dev.md`, `QA.md`를 교차 검증하여 보고서와 개발 문서 일관성을 확보합니다。
* 모든 코드 작성 및 수정 작업은 해당 파트의 MD 문서에 반영되어야 하며, PM에게 공유하여 기록 누락을 방지합니다.
* 모든 코드 변경은 충분한 주석과 설명을 포함해야 하며, 향후 고도화 및 기능 개선 시 코드 리뷰 비용을 줄이는 것이 목표입니다.
* 보고서는 기술 구현 결과뿐 아니라 진행 상황, 리스크 대응, 역할별 누락 체크를 함께 기록하여 발표 준비에 도움을 줍니다.
* 보고서 업데이트 항목:
    - 최신 QA 통과 현황
    - 병합 충돌 및 역할 분리 이슈 해결
    - 미구현 기능 및 시연 예외 안내 메시지

## 3. 주요 기능 및 AI 활용 전략
* **주요 기능**: 사용자 프로필 관리, 커리어 타임라인 생성, 이력서 작성/저장, AI 코칭, 관리자 대시보드, 문서 업로드/통계 조회.
* **AI 활용 (Co-pilot & Agent 활용)**:

    - **데이터 최적화**: 시니어 타겟 유저를 위한 문서 파싱 가이드 프롬프트 설계.
    - **디버깅**: 에이전트를 통한 실시간 에러 로그 분석 및 QA 피드백 반영.

## 5. 주요 개발 이슈 및 해결 사례

### 5.1. 3일차 QA: 치명적 병합 충돌(Merge Conflict) 해결
*   **이슈 요약**: 3일차 QA 과정에서 사용자 대시보드(`dashboard.html`)와 관리자 페이지(`admin.html`)의 코드가 뒤섞이는 심각한 Git 병합 충돌이 발견되었습니다. 이로 인해 두 페이지 모두 정상 기능이 불가능하여 시연에 치명적인 결함으로 작용했습니다.
*   **AI 에이전트 활용**: QA 에이전트의 시나리오 기반 테스트 중 문제가 식별되었고, 개발 에이전트가 즉시 코드 분석에 착수했습니다. AI는 병합 과정에서 발생한 코드 중첩 및 누락 부분을 특정하고, 각 파일의 역할(사용자/관리자)에 맞는 코드 블록을 제안했습니다.
*   **해결 과정**:
    1.  **`dashboard.html`**: 관리자 전용 통계 UI 및 관련 JavaScript 로직을 제거하고, 순수 사용자 대시보드 기능으로 복원했습니다.
    2.  **`admin.html`**: 불필요한 로그인 로직과 사용자 페이지로의 리디렉션 코드를 제거하여 관리자 전용 페이지로 정상화했습니다.
*   **결과 및 교훈**: AI 에이전트를 활용한 신속한 QA 및 디버깅 프로세스를 통해 MVP 시연에 큰 위협이 될 수 있었던 문제를 조기에 해결했습니다. 이는 `pm.md`에 정의된 리스크 관리 및 QA 피드백 반영 절차가 효과적으로 작동했음을 보여주는 사례입니다. 또한, 향후 유지보수성 향상을 위해 각 페이지의 JavaScript를 별도 파일로 분리하는 작업의 필요성이 대두되었습니다.
