## 5. 주요 개발 이슈 및 해결 사례

### 5.1. 3일차 QA: 치명적 병합 충돌(Merge Conflict) 해결
*   **이슈 요약**: 3일차 QA 과정에서 사용자 대시보드(`dashboard.html`)와 관리자 페이지(`admin.html`)의 코드가 뒤섞이는 심각한 Git 병합 충돌이 발견되었습니다. 이로 인해 두 페이지 모두 정상 기능이 불가능하여 시연에 치명적인 결함으로 작용했습니다.
*   **AI 에이전트 활용**: QA 에이전트의 시나리오 기반 테스트 중 문제가 식별되었고, 개발 에이전트가 즉시 코드 분석에 착수했습니다. AI는 병합 과정에서 발생한 코드 중첩 및 누락 부분을 특정하고, 각 파일의 역할(사용자/관리자)에 맞는 코드 블록을 제안했습니다.
*   **해결 과정**:
    1.  **`dashboard.html`**: 관리자 전용 통계 UI 및 관련 JavaScript 로직을 제거하고, 순수 사용자 대시보드 기능으로 복원했습니다.
    2.  **`admin.html`**: 불필요한 로그인 로직과 사용자 페이지로의 리디렉션 코드를 제거하여 관리자 전용 페이지로 정상화했습니다.
*   **결과 및 교훈**: AI 에이전트를 활용한 신속한 QA 및 디버깅 프로세스를 통해 MVP 시연에 큰 위협이 될 수 있었던 문제를 조기에 해결했습니다. 이는 `pm.md`에 정의된 리스크 관리 및 QA 피드백 반영 절차가 효과적으로 작동했음을 보여주는 사례입니다. 또한, 향후 유지보수성 향상을 위해 각 페이지의 JavaScript를 별도 파일로 분리하는 작업의 필요성이 대두되었습니다.

### 5.2. 신규 기능 검증: 프로필 이미지 업로드 (QA PASS)
*   **이슈 요약**: 신규 구현된 '사용자 프로필 이미지 업로드' 기능이 MVP 기준 및 보안 정책에 맞게 동작하는지 QA를 진행했습니다.
*   **검증 결과 및 반영 사항**:
    1.  **보안/예외 처리**: 파일명 UUID 난수화, 허용 확장자 화이트리스트 검사, 업로드 디렉토리 부재 시 자동 생성 등 필수 안전장치 동작 확인 완료.
    2.  **UX 최적화**: 업로드 후 즉각적인 UI 반영을 가로막는 브라우저 캐싱 문제를 타임스탬프(`?t=timestamp`) 기법으로 우회한 점 확인 완료.
*   **결과**: MVP 목적에 완벽하게 부합하며 안정성이 보장된 훌륭한 코드로 판정되어 정식 시연 스펙으로 확정되었습니다.

### 5.3. 어드민 기능 고도화: 템플릿 파일 업로드 및 미리보기
*   **이슈 요약**: 기존 텍스트(HTML) 입력 방식의 템플릿 등록은 관리자 편의성이 낮고 실제 문서와의 괴리가 발생할 위험이 있었습니다.
*   **해결 방안**: PDF/WORD 파일을 직접 업로드하고 관리하는 시스템으로 전환했습니다.
*   **성과**:
    - **파일 기반 관리**: 실제 이력서 파일을 템플릿으로 등록하여 출력물 품질 보장.
    - **인쇄 및 미리보기**: 브라우저 내장 기능을 활용한 즉각적인 미리보기 및 인쇄 다이얼로그 호출 기능 구현.
    - **자동화**: 템플릿 삭제 시 서버 내 물리 파일도 동시 삭제되도록 로직 강화.

### 5.4. 보안 및 인프라 표준화
*   **환경 변수**: `.env.example` 도입으로 팀원 간 설정 공유를 표준화하고, `app.py`에서 자동으로 로드하도록 개선하여 배포 편의성을 높였습니다.
*   **보안**: `.gitignore` 최적화를 통해 개발 중 생성되는 임시 파일 및 DB 파일의 유출을 원천 차단했습니다.

### 5.5. UX/UI 일관성 강화: 커스텀 토스트 알림 도입 (진행 중)
*   **이슈 요약**: QA 과정에서 미구현 기능 안내, 저장 성공/실패 등 사용자에게 피드백을 주는 방식이 페이지마다 달라 일관성이 부족하다는 점이 지적되었습니다. 특히, `builder.html` 등 일부 페이지에서는 여전히 시스템 기본 `alert()` 창이 사용되어 시연 시 전문성을 저해할 위험이 있었습니다.
*   **해결 방안**:
    1.  **커스텀 토스트(Toast) UI 도입**: `timeline.html`, `dashboard.html`에 우선 적용된 세련된 디자인의 토스트 알림 컴포넌트를 모든 페이지의 표준으로 채택했습니다.
    2.  **`handleUnavailableFeature()` 함수 개선**: 기존 `alert()`를 호출하던 로직을 `showToast()` 함수를 호출하도록 변경하여, "준비 중인 기능입니다" 안내 메시지를 일관된 UI로 제공합니다.
*   **기대 효과**: 모든 사용자 피드백 메시지가 통일된 디자인으로 제공되어 데모 시연의 완성도와 전문성이 크게 향상됩니다.

### 5.7. Gemini 손상 복구 — 전 사이트 CSS 붕괴 및 어드민 중복 렌더링 (2026-06-24)
*   **이슈 요약:** Gemini(VSCode)가 두 가지 치명적 손상을 유발했습니다.
    1.  **BUG-004 (admin.html 중복 렌더링):** Gemini가 `_admin_sidebar.html`, `_admin_header.html`을 분리 생성하면서 `admin.html` 원본 코드를 삭제하지 않아 사이드바·헤더가 각 2개씩 이중 렌더링됨.
    2.  **BUG-005 (전 사이트 CSS 붕괴):** Gemini가 `_head.html` 편집 중 `tailwind.config` 스크립트 블록 전체를 삭제. `_head.html`은 모든 페이지에 `{% include %}`로 공유되므로 전 페이지의 커스텀 색상·폰트·여백이 완전히 무효화됨 (버튼·로고·제목 등 디자인 전체 붕괴).
*   **복구 방법:**
    1.  `admin.html`: `{% include %}` 태그 이후 중복 하드코딩된 HTML 블록 제거.
    2.  `_head.html`: 팀원 Replit 원본 사이트(`pike.replit.dev`)에서 `tailwind.config` 블록을 `curl`로 직접 추출하여 복원.
*   **검증 결과:**
    -   admin 페이지: `<aside>` 1개, `<header>` 1개, 3개 사용자 경로(일반/데모/관리자) 서버사이드 검증 전건 PASS.
    -   CSS 복원: main.html 스크린샷으로 참고 사이트 대비 시각적 완전 일치 확인.
*   **교훈 및 통제 강화:**
    -   `_head.html` 등 공유 파티얼(partial) 파일은 Gemini 수정 금지. Replit Agent 전담.
    -   `_admin_sidebar.html`, `_admin_header.html` 등 공통 컴포넌트 분리 작업도 Replit Agent 전담.
    -   PM 통제: 모든 작업은 착수 즉시 `pm.md`에 등록하고, 완료 즉시 전 역할 문서(`PROGRESS.md`, `UI.md`, `dev.md`, `QA.md`, `report.md`) 동기 업데이트 의무화.

### 5.6. 시연 안정성 강화: 파일 업로드 예외 처리
*   **이슈 요약**: `select.html`의 이력서 파일 업로드 기능에서 허용되지 않는 파일 형식(예: `.jpg`, `.zip`)을 올릴 경우, 적절한 사용자 안내 없이 스크립트 에러가 발생하며 기능이 멈추는 치명적 결함이 발견되었습니다.
*   **해결 방안**:
    1.  **프론트엔드 검증**: 파일 선택 시 `accept` 속성(`.pdf,.doc,.docx`)을 명시하고, JavaScript 단에서 파일 확장자를 한 번 더 검증하여 허용되지 않는 파일은 업로드 시도를 차단합니다.
    2.  **사용자 피드백**: 잘못된 파일을 올릴 경우, 새로 도입된 커스텀 토스트 알림을 통해 "지원하지 않는 파일 형식입니다."라는 명확한 에러 메시지를 사용자에게 보여줍니다.
*   **결과**: 시연 중 발표자가 실수로 다른 파일을 업로드하더라도 데모가 중단되지 않고 안정적으로 다음 단계로 넘어갈 수 있도록 방어 로직이 강화되었습니다.
