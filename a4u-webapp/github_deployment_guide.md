# a4u Design Export & GitHub Deployment Guide

Since you've created a comprehensive career management platform with Stitch, here is how to export your designs and host them on GitHub Pages correctly.

## 1. Exporting Your Screens
For each screen on your canvas (e.g., Main Landing Page, Dashboard):
1.  Click the **Export** button in the top toolbar.
2.  Select **Download HTML/CSS**.
3.  You will receive a ZIP file containing:
    *   `index.html` (The structure)
    *   `style.css` (The Executive Kinetic theme)
    *   An `assets/` folder (Contains images like logos and persona photos)

## 2. Resolving Image Path Issues
The primary reason images break on GitHub is **absolute vs. relative paths**.

### The Problem
Stitch uses local placeholders like `{{DATA:IMAGE:IMAGE_1}}` during design, which are converted to absolute URLs during export. When you move these files to GitHub, the HTML might still look for images in a fixed location that doesn't exist in your repo.

### The Fix: Relative Pathing
1.  **Organize your folder:**
    ```text
    /your-repo-name
    ├── index.html
    ├── dashboard.html
    ├── style.css
    └── /assets
        ├── logo.png
        └── profile.jpg
    ```
2.  **Update HTML `<img>` tags:**
    Open your `.html` files and ensure your image sources point to the local `assets` folder:
    *   *Change:* `<img src="https://stitch-assets.../logo.png">`
    *   *To:* `<img src="assets/logo.png">`

## 3. GitHub Pages Deployment
1.  **Create a Repository:** Go to GitHub and create a new repository (e.g., `a4u-prototype`).
2.  **Upload Files:** Use the "Upload files" button to drag and drop your `index.html`, `style.css`, and the `assets` folder.
3.  **Enable Pages:**
    *   Go to **Settings** > **Pages**.
    *   Under **Branch**, select `main` and `/ (root)`.
    *   Click **Save**.
4.  **Live URL:** Your site will be live at `https://[your-username].github.io/[repo-name]/`.

## 4. Pro-Tip: Consolidating Multiple Screens
If you exported 4 separate screens:
*   Rename them to `index.html`, `dashboard.html`, `builder.html`, and `timeline.html`.
*   Open the code and update the navigation links (`<a>` tags) to point to these local filenames (e.g., `<a href="dashboard.html">`).
